# PGR Monthly Decision Log

Automated recommendations generated by `scripts/monthly_decision.py` on the 20th of each month
(or the first business day after the 20th if the 20th falls on a weekend).

Each run produces a dated subfolder under `results/monthly_decisions/YYYY-MM/` containing:
- `recommendation.md` — Full sell/hold report with tax impact
- `signals.csv` — Per-benchmark signals (ticker, IC, hit_rate, predicted_return, signal)
- `backtest_summary.csv` — Monthly stability stats at this point in time
- `plots/` — IC time series, regime breakdown visualizations

## Column Definitions

| Column | Description |
|--------|-------------|
| As-Of Date | The date the model uses as its information cutoff (typically the 20th) |
| Run Date | When the script actually executed |
| Consensus Signal | OUTPERFORM / UNDERPERFORM / NEUTRAL (majority across all benchmarks) |
| Sell % | Recommended percentage of vesting RSUs to sell (Kelly-based in v3.1+) |
| Predicted 6M Return | Weighted-average 6-month PGR relative return prediction |
| Mean IC | Average information coefficient across active benchmarks |
| Hit Rate | Average directional accuracy across active benchmarks |
| Notes | Any anomalies, data gaps, or model warnings for this run |

## Interpreting the Recommendation

- **OUTPERFORM + Sell % < 50%**: Model has high conviction PGR will outperform benchmarks over the next 6 months. Hold the majority of shares.
- **NEUTRAL**: Model signal is weak (IC < 0.05). Default to 50% diversification sale regardless of direction.
- **UNDERPERFORM + Sell % ≥ 75%**: Model predicts PGR will underperform its benchmarks. Diversify most of the vesting tranche.

The sell % recommendation is used only at actual vesting events (January and July). Monthly runs provide ongoing signal quality monitoring and early warning of trend changes.
